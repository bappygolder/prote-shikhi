## Findings (non-blocking notes — none of these are FAIL)

These are observations a future reader might want to know. None block CTX-06.

1. **Schema field-name drift: `seenCount` vs spec's `attempts`.** The spec (§4) names the cumulative-shows counter `attempts`; the code keeps the v1 name `seenCount`. Same semantic ("total times shown") and `applyGrade` increments it on both correct and wrong (`learning.ts:117`). This is a knowingly-deferred rename — keeping it minimizes the v1→v2 migration footprint. Worth aligning before the spec freezes, but explicitly out of scope for CTX-05's "schema additive" charter.

2. **Spec-extra fields not yet incremented.** `attemptsSinceEnteringActive`, `enteredActiveAt`, `cardsShownSinceMastered`, and `sprinkleCooldown` exist in `LetterProgress` and migrate cleanly, but `applyGrade` does not yet update them — they stay at their default for live cards. This is correct: those fields are driven by the active-set lifecycle and selection logic that CTX-06 will land. The migration sets `cardsShownSinceMastered: 999` for already-mastered v1 cards (so the newly-mastered quiet-period rule does not retroactively fire), which is a sound default.

3. **Reviewer could not perform the browser smoke test.** Mitigated by code-path inspection (App.tsx writes the v2 envelope unconditionally) and unit-test coverage. Bappy may want to do one human pass: open the app, grade a card, hard-reload, open DevTools → Application → Local Storage → `bornomala.progress.v1`, and confirm the shape is `{"schemaVersion":2,"byCard":{...}}`.

4. **Uncommitted working-tree change.** `App.tsx` has a pending diff bumping `APP_VERSION` from `v1.1.1` to `v1.1.2` and `LAST_UPDATED` to `Wednesday, 6 May 2026`. This is unrelated to CTX-05 (purely metadata) and is already noted in the prior session memory. Not part of this review's scope; commit or revert before starting CTX-06 to avoid mixing it into the next slice.
